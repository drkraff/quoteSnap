## Screens in Scope

Phase 2 introduces one new screen flow inserted between registration success and the main app shell. The flow is linear (no back navigation once started):

1. **Trade Selection screen** — Single-choice grid of trade cards (Plumbing, Electrical, HVAC). Contractor taps one card; selection state is highlighted. CTA enabled only after a trade is selected.
2. **Seeding screen** — Full-screen progress indicator. Shown while catalog seeds from server (or falls back to bundled template if offline). No interaction required; auto-advances on completion.
3. **Ready screen** — Confirmation that catalog is ready. Single CTA advances to main app.

The existing `/(auth)/login.tsx` and `/(auth)/register.tsx` screens are already built; do not re-specify their contract here.

## Interaction Contract

### Trade Selection
- Default state: all 3 cards unselected; CTA button `disabled={true}` (grayed, no press feedback)
- Selected state: tapped card receives secondary background + accent border (2px) + checkmark overlay (Unicode `\u2713` in accent color, top-right corner, 14px)
- Only one card selectable at a time; tapping a second card deselects the first
- CTA activates (`disabled={false}`) immediately on first valid selection
- Minimum tap area for each trade card: 44px height (enforced via `minHeight: 44` in StyleSheet)

### Seeding Screen
- Displayed immediately after CTA tap on Trade Selection; no loading delay before showing
- `ActivityIndicator size="large"` centered; status text below reads "Setting up your catalog..."
- If server catalog fetch succeeds: advance to Ready screen automatically (no user action)
- If offline or fetch fails within 5 seconds: show amber notice "No signal — using starter catalog. You can update this later." Status text changes to "Loading starter catalog..." Auto-advances after 1 additional second
- No back navigation available on this screen (prevent partial setup)

### Ready Screen
- Auto-advances is NOT used here; contractor must tap CTA to enter the app (intentional moment of orientation)
- CTA label: "Start Quoting" (see Copywriting Contract)

## Accessibility Contract

- All Pressable components include `accessibilityRole="button"` and `accessibilityLabel` matching visible text
- TradeCard selected state includes `accessibilityState={{ selected: true }}`
- ActivityIndicator includes `accessibilityLabel="Loading, please wait"`
- Minimum touch targets: 44px height on all interactive elements
- Color is never the sole indicator of state: selected trade card uses border + background + checkmark (not color alone)
