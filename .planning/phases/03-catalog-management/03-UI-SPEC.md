# Phase 3 — UI Design Contract

> Visual and interaction contract for Phase 3: Catalog Management. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (React Native StyleSheet.create) |
| Preset | not applicable |
| Component library | React Native core (View, Text, Pressable, SectionList, TextInput, Modal) |
| Icon library | @expo/vector-icons (Ionicons) — already available in Expo SDK |
| Font | System default (no custom font loaded in existing screens) |

**Rationale:** This is a React Native / Expo app. shadcn is web-only and does not apply. The existing pattern is inline StyleSheet.create per file. Phase 3 follows that same pattern. A shared constants file (`src/theme/tokens.ts`) should be introduced to hold the values declared in this spec — prevents value drift across screens.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, unit badge internal padding, checkmark positioning |
| sm | 8px | Row gap between list items, gap between form fields, card inner padding |
| md | 16px | Screen horizontal padding, button vertical padding, section header padding |
| lg | 24px | Screen top padding, bottom sheet top padding |
| xl | 32px | Space above primary CTA at bottom of screen |
| 2xl | 48px | Bottom sheet handle-to-content gap |
| 3xl | 64px | Bottom sheet minimum bottom inset (safe area) |

Exceptions:
- Touch targets: minimum 44px height on all interactive rows, buttons, and swipe action areas — non-negotiable for gloved hands on-site (source: CONTEXT.md specifics)
- Bottom sheet drag handle: 4px tall, 32px wide — not a spacing token, it is a fixed dimension

|------|-------|-------|
| Dominant (60%) | `#ffffff` | Screen background, bottom sheet background, list row background |

| Accent (10%) | `#0066cc` | Primary CTA button fill, Save Item button fill, active tab indicator, swipe-archive icon tint on press |
| Destructive | `#dc2626` | Swipe-to-archive revealed action background (red), destructive icon |
| Warning | `#b45309` | Offline sync-pending indicator dot only |
| Muted text | `#666666` | Subheadings, placeholder text, secondary metadata |
| Border | `#cccccc` | List row bottom border, form field border (default state) |
| Border focused | `#0066cc` | Form field border when input is focused |
| Error text | `#dc2626` | Inline validation error messages below fields |

Accent (`#0066cc`) reserved for:
1. Primary CTA button ("Add Item") fill
2. Save Item button fill in bottom sheet
3. Active tab indicator in bottom tab bar
4. Swipe-archive icon tint on reveal (icon only, not background — background is destructive red)

Accent is NOT used on: list rows, section headers, unit badges, prices, or any passive UI element.

Source: accent and background values extracted from `trade-selection.tsx` and `seeding.tsx`. Warning amber `#b45309` from `seeding.tsx` offlineNotice.

## Interaction Contracts

### Catalog list

- `SectionList` with `stickySectionHeadersEnabled={true}`.
- Each `CatalogRow` is swipeable left to reveal archive action using `react-native-gesture-handler` `Swipeable` (already in Expo SDK).
- Archive action: red background (#dc2626), white trash icon, full-height action area (same height as row, minimum 44px).
- Swipe reveal does not immediately archive — it reveals the action button. Tapping the action button triggers archive.
- After archive: row animates out, UndoToast appears. Undo restores row in-place. Toast auto-dismisses at 3s.
- Tapping a row opens `ItemFormSheet` in edit mode. Row tap target is full-width, minimum 44px height.

### Add/Edit bottom sheet

- Opens as a Modal with `animationType="slide"` from bottom. Not a third-party bottom sheet library — use RN Modal to avoid adding a dependency.
- Sheet is full-width, auto-height based on content, maximum 80% of screen height.
- Drag handle: 4px tall, 32px wide, `#cccccc`, centered, 12px from top of sheet.
- Fields render top to bottom: Name → Unit → Price.
- Unit field: rendered as a row of 5 pressable pill buttons (each / hour / foot / sqft / job). No external picker. Selected pill: `#0066cc` background, white text. Unselected: `#f5f5f5` background, `#333` text.
- Price field: TextInput with `keyboardType="decimal-pad"`. Displays formatted dollars (e.g., `$125.00`). On change: strip non-numeric, store internally as cents integer. Prefix `$` is a non-editable Text sibling rendered left of the TextInput.
- Validation: inline error Text (14px, `#dc2626`) appears below each invalid field on first submit attempt. Re-validates on each subsequent change.
- Save Item button: full-width, 44px height, `#0066cc` fill, white label "Save Item". Disabled (opacity 0.5) until all three fields are valid.
- Cancel: `×` icon (Ionicons `close`, 24px) in top-right corner of sheet. No confirmation.
- Keyboard: sheet scrolls up when keyboard appears (use `KeyboardAvoidingView` wrapping the sheet content).

### Tab bar

- `Tabs` navigator from expo-router (wraps React Navigation Bottom Tab Navigator).
- Two tabs: "Home" (Ionicons `home-outline` / `home`) and "Catalog" (Ionicons `list-outline` / `list`).
- Active tab: icon solid variant, label `#0066cc`. Inactive tab: outline variant, label `#666666`.
- Tab bar background: `#ffffff`. Top border: 1px `#cccccc`.
- Tab label: 12px, weight 400 — below icon. Not part of the 4-size typography scale (tab labels are navigation chrome, not content).

### FAB (Floating Action Button)

- Positioned bottom-right of catalog screen, 24px from right edge, 24px above bottom safe area.
- `#0066cc` fill, white `+` icon (Ionicons `add`, 28px), 56px diameter circle, shadow: `elevation: 4` (Android) / `shadowColor: '#000', shadowOffset: {width:0, height:2}, shadowOpacity: 0.2, shadowRadius: 4` (iOS).
- Tapping opens `ItemFormSheet` in add mode.

### Offline sync indicator

- Single amber dot (`#b45309`, 8px diameter circle) rendered in the top-right of the Catalog screen header when there are unsynced items in the sync queue.
- No label — dot only. Accessible via `accessibilityLabel="Catalog has unsynced changes"`.
- Disappears when sync queue is empty.

## Accessibility Contract

- All interactive elements have `accessibilityRole` and `accessibilityLabel`.
- `CatalogRow`: `accessibilityRole="button"`, label includes item name and price (e.g., "Pipe repair, $125.00 per each. Double tap to edit.").
- Archive swipe action: `accessibilityRole="button"`, `accessibilityLabel="Archive {item name}"`.
- Unit picker pills: `accessibilityRole="radio"`, `accessibilityState={{ checked: isSelected }}`.
- Save Item button: `accessibilityState={{ disabled: !isValid }}`.
- UndoToast: `accessibilityLiveRegion="polite"` so screen readers announce it.
- Minimum touch target 44px on all interactive elements — enforced via `minHeight: 44`.
