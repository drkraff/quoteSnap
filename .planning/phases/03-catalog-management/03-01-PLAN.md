    id: string;

  Create `apps/backend/src/routes/catalog.ts` with an Express Router exported as `router`. Follow the onboarding.ts pattern exactly (import Router, Request, Response from "express", import authenticateToken from "../middleware/auth.js", import query from "../db/connection.js").

  All 4 endpoints use `authenticateToken` middleware and extract `contractorId` from `req.contractor!.contractorId`.

  **GET / (list active items):**
  - SQL: `SELECT id, name, unit, unit_price_cents, trade_category, is_archived, created_at, updated_at FROM catalog_items WHERE contractor_id = $1 AND is_archived = FALSE ORDER BY trade_category ASC NULLS LAST, name ASC`
  - Map rows: snake_case columns to camelCase CatalogItemResponse. `created_at` and `updated_at` as ISO strings.
  - Return: `res.json({ items: CatalogItemResponse[] })`

  **POST / (create item) — per D-06, D-08:**
  - Body: `CreateCatalogItemBody`
  - Validate: `name` must be non-empty string (trimmed), `unitPriceCents` must be integer > 0, `unit` must be one of `['each', 'hour', 'foot', 'sqft', 'job']`.
  - Return 400 with `{ error: string }` for each validation failure (check name first, then unit, then price).
  - SQL: `INSERT INTO catalog_items (contractor_id, name, unit, unit_price_cents, trade_category) VALUES ($1, $2, $3, $4, $5) RETURNING id, name, unit, unit_price_cents, trade_category, is_archived, created_at, updated_at`
  - Return: `res.status(201).json({ item: CatalogItemResponse })`

  **PUT /:id (update item) — per D-06, D-08:**
  - Body: `UpdateCatalogItemBody` (all fields optional but at least one required)
  - Validate: if `name` provided, must be non-empty. If `unitPriceCents` provided, must be integer > 0. If `unit` provided, must be one of the valid units.
  - Return 400 if no fields provided (`{ error: "At least one field required" }`).
  - SQL: Build dynamic SET clause from provided fields only. Include `WHERE id = $N AND contractor_id = $M` to prevent cross-contractor access.
  - Return 404 `{ error: "Item not found" }` if no rows updated.
  - Return: `res.json({ item: CatalogItemResponse })` using a subsequent SELECT or RETURNING.

  **PATCH /:id/archive — per D-10:**
  - SQL: `UPDATE catalog_items SET is_archived = TRUE WHERE id = $1 AND contractor_id = $2 AND is_archived = FALSE RETURNING id`
  - Return 404 if no rows affected.
  - Return: `res.json({ archived: true })`

  All endpoints wrapped in try/catch returning 500 on unexpected errors. Use `console.error` for server-side logging (matching onboarding.ts pattern).

  VALID_UNITS constant: `const VALID_UNITS = ['each', 'hour', 'foot', 'sqft', 'job'] as const;` — per D-06.
