## Spacing Scale

Source: `apps/mobile/src/theme/tokens.ts` — confirmed match to 8-point scale.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Row internal padding, stepper gap |
| md | 16px | Screen horizontal padding, list item vertical padding |
| lg | 24px | Section header spacing, sheet padding |
| xl | 32px | Large section breaks |
| 2xl | 48px | Major layout gaps |
| 3xl | 64px | Page-level bottom padding (clears FAB + sticky footer) |

Exceptions:
- `MIN_TOUCH_TARGET = 44px` — minimum height/width for all interactive elements. Mandatory on +/- stepper buttons, Send button, Delete swipe action, status badge rows, and price tap target. Source: `tokens.ts` + CONTEXT.md specifics (gloved hands).
- Sticky footer padding-bottom: 34px (safe area inset) + 16px — accounts for home indicator on modern Android/iOS. Use `useSafeAreaInsets()`.
- FAB: 56px diameter, 16px from right edge, 16px above sticky footer top edge.

## Typography

Source: `apps/mobile/src/theme/tokens.ts` — three sizes already defined. Phase 4 uses all three plus adds one display size for quote totals.

**Weights in use: 400 (regular) and 700 (bold) — exactly 2 weights.**

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Label | 14px | 400 | 20px (1.43) | Status badge text, relative date ("2 days ago"), unit labels on line item rows, sheet field labels |
| Body | 16px | 400 | 24px (1.5) | Line item name, customer phone number display, catalog picker item names, history row phone number |
| Display | 20px | 700 | 26px (1.3) | Line item price (tappable), history row total price, draft subtotal in sticky footer |
| Heading | 22px | 700 | 28px (1.27) | Screen titles ("New Quote", "Quote History"), sheet titles ("Edit Price", "Add Item") |

Notes:
- Display (20px/700) is a new token for this phase. Add to `tokens.ts` as `typography.display`. Raised from 18px to 20px to create a readable 4px gap above Body (16px) on mobile.
- Weight 600 is not used in this phase. Only 400 and 700 are in the contract.
- The existing three tokens (label 14px/400, body 16px/400, heading 22px/700) are unchanged.

## Interaction Contract

### FAB
- Position: bottom-right, 16px from right edge, 16px above sticky footer top (on draft screen) or 16px above safe area bottom (on history screen)
- Size: 56px diameter
- Icon: `Ionicons add` (24px, white)
- Background: `colors.accent` (#0066cc)
- Accessibility: `accessibilityLabel="Create new quote"` on the TouchableOpacity/Pressable wrapping the FAB
- Behavior: Creates new WatermelonDB `quote` (status: draft_local) + empty `draft` record, navigates to `app/(app)/draft/[id]`
- Hidden on: quote detail screen (read-only)

### +/- Stepper (LineItemRow)
- Each button: minimum 44x44px touch target (critical — gloved hands)
- `-` button: left of quantity display. Disabled at quantity = 1 (never zero). Opacity 0.4 when disabled.
- `+` button: right of quantity display.
- Quantity display: 20px wide min, body text (16px/400), centered
- Interaction: tap fires `database.write()` immediately (REVIEW-05). No debounce.

### Price tap (LineItemRow)
- Tappable area: entire price cell, minimum 44px height
- Opens PriceEditSheet modal
- Pre-filled with current price (dollar-formatted)
- "Update Price": `database.write()` immediately, dismiss modal
- "Discard": dismiss modal, no write

### Swipe-to-delete (LineItemRow)
- Direction: swipe left
- Reveals: red Delete action (background `colors.destructive`)
- Label: "Delete" in white, label text 14px/400
- Confirm: no confirmation modal — immediate delete on action tap (consistent with catalog swipe-to-archive behavior). UndoToast appears for 4 seconds.
- Write: `database.write()` immediately on delete

### Send button (sticky footer)
- Height: 48px, full width minus 32px horizontal margin
- Background disabled: `colors.secondary` (#f5f5f5)
- Background enabled: `colors.accent` (#0066cc)
- Label: "Send Quote" (16px/700, white when enabled, mutedText when disabled)
- Disabled conditions: zero line items OR invalid phone number
- On tap while disabled: inline error appears below phone input (not toast). See Copywriting Contract.

### Status badge (QuoteRow, QuoteDetail)
- Shape: rounded pill, 4px border-radius
- Padding: 8px horizontal, 4px vertical — uses `sm` (8px) and `xs` (4px) tokens; fits 14px badge text
- Font: 14px/700 — uses Label size at bold weight for badge readability; uses bold weight from the 2-weight contract
- Color map (background / text):
  - `draft_local`: `#e0f0ff` / `#0066cc` (light accent tint / accent)
  - `draft_queued`: `#e0f0ff` / `#0066cc`
  - `sent`: `#f0f0f0` / `#666666`
  - `approved`: `#dcfce7` / `#16a34a` (light green tint / success)
  - `declined`: `#fee2e2` / `#dc2626` (light red tint / destructive)
  - `expired`: `#fee2e2` / `#dc2626`
  - `failed_send`: `#fee2e2` / `#dc2626`
- Display labels: "Draft" | "Queued" | "Sent" | "Approved" | "Declined" | "Expired" | "Failed"

### Navigation transitions
- Quotes tab: standard Expo Router tab switch (no animation customization)
- Draft review screen: `router.push('/(app)/draft/' + id)` — slide from right (default stack behavior)
- Quote detail screen: `router.push('/(app)/quote/' + id)` — slide from right
- Back: hardware back / swipe-back (default Expo Router behavior)

## States Reference

### History list screen
| State | Behavior |
|-------|----------|
| Has quotes | FlatList renders QuoteRow items |
| Empty | EmptyState component (heading + body + icon) |
| Loading | Not needed — WatermelonDB observe() delivers synchronously |

### Draft review screen
| State | Behavior |
|-------|----------|
| Empty draft (no line items) | EmptyState in list area, "Add Item" button below, Send button disabled |
| Has items, invalid phone | Send button disabled, no inline error until tap attempt |
| Has items, valid phone | Send button enabled (accent color) |
| Tap Send while invalid | Inline error message appears below phone input (no toast) |
| Tap Send while valid | Triggers send flow (Phase 6 wires this; Phase 4 queues to sync) |
| Item deleted | UndoToast for 4 seconds |

### Quote detail screen (read-only)
| State | Behavior |
|-------|----------|
| Any non-draft status | Read-only QuoteDetail component, no FAB, no edit controls |
